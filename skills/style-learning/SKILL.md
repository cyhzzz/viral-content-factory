# 风格学习流程

## 核心定位

从用户提供的满意文章中，深度提取写作风格特征，生成「作者风格手册」供多平台创作使用。

---

## 完整工作流

```
Phase 1  文章采集存档 ✋
    ↓
Phase 2  多维度风格分析
    ↓
Phase 3  一致率分析 → 风格画像
    ↓
Phase 4  多平台风格生成 ✋
    ↓
Phase 5  System Prompt注入 + 风格手册定稿 ✋
```

---

## Phase 1: 文章采集存档 ✋

> **强制优先原则**：风格必须来自你本人发布的真实文章，不使用预设人格模板。
> 1 篇即可提取，3 篇以上提取更完整。

### 询问用户

```
请提供你最满意的文章（1篇起，3篇以上更完整）。
支持以下方式：
- 粘贴文本（直接粘贴文章内容）
- 上传文件（.md / .txt）
- 指定URL（公众号/知乎/微博链接）

来源平台：{平台名称}
作者名：{作者名}
```

### 素材验证

| 文章数量 | 处理方式 |
|:---:|:---|
| ≥3 篇 | 完整提取（8+11 维度） |
| 1-2 篇 | 轻量提取（已识别的维度提取，未明确的标注 `[待补充]`） |
| 0 篇 | 引导用户提供；用户拒绝 → 降级人格，明确警告准确度有限 |

> ⚠️ **人格降级警告**：如用户拒绝提供文章，将使用预设人格模板（midnight-friend / warm-editor 等）。这是通用模板，**不是你本人的风格**，文章辨识度和人感会显著降低。请优先说服用户提供至少 1 篇真实文章。

### 存档

```bash
# URL抓取
python3 {skill_dir}/scripts/fetch_article.py <url> -o {skill_dir}/references/exemplars/article_{N}.md

# 本地保存
# 文章存入 {skill_dir}/references/exemplars/
```

### Phase 1 确认点

展示采集到的文章列表，请用户确认：
- 文章列表和字数
- 来源平台和作者是否正确
- 是否可以开始提取（1篇起即可，3篇以上更完整）

**最低要求**：1 篇。如用户仅提供 1 篇，告知将执行轻量提取模式。

## Phase 2: 多维度风格分析

### 分析框架

```
读取: skills/style-learning/references/style-extract.md
读取: skills/style-learning/references/author-profile.md
```

### 8维度深度提取（财经场景）

| 序号 | 维度 | 提取内容 | 权重 |
|:---:|:---|:---|:---:|
| 1 | 标题格式 | 主标题结构、副标题格式、栏目名称 | 20% |
| 2 | 段落结构 | 段落数量、每段功能定位、段落间衔接逻辑 | 20% |
| 3 | 句式特征 | 句子长度偏好、条件判断句、概率表达 | 15% |
| 4 | 词汇特征 | 专业术语库、特色比喻词、谨慎词使用 | 15% |
| 5 | 分析逻辑 | 数据→事件→逻辑→结论→建议框架 | 20% |
| 6 | 数据规范 | 数据类型、精确度要求、对比表达、来源标注 | 参考 |
| 7 | 固定表达 | 段落开头固定语、结尾固定格式、品牌呼号 | 10% |
| 8 | 数据需求 | 必需数据、可选数据、资讯需求 | 参考 |

### 作者定位识别

```
读取: skills/style-learning/references/author-profile.md
```

**提取内容**：
- 身份标签（分析师/自媒体/机构/个人）
- 内容定位（专业向/小白向/混合）
- 读者画像（谁在读这篇文章）
- 平台差异（跨平台录入时，分析同一作者在不同平台的风格差异）

### 11维度深度提取（小红书场景，额外）

| 序号 | 维度 | 提取内容 |
|:---:|:---|:---|
| 9 | 段落强制开头语 | 每段必须使用的固定开头语 |
| 10 | 比喻体系 | 核心比喻词 + 指代含义 |
| 11 | 图片使用 | 图片数量偏好、封面/尾页结构 |

## Phase 3: 一致率分析 → 风格画像

### 一致率规则

| 一致率 | 类型 | 处理方式 |
|:---|:---|:---|
| ≥80% | 核心模式 | 进入风格画像 |
| 50-79% | 标准模式 | 标记为可选变体 |
| <50% | 个人偏好 | 作为个性化选项 |

```
【风格学习完成】✋

📱 微信长文
- 语气正式度：3.5/5（比原文稍正式）
- 段落结构：长段落为主，平均300字
- 数据呈现：精确到小数点后2位
- 风险提示：每篇结尾必含
- 调整点：新增「数据→结论」的论证链条强化

- 段落结构：单段落，140字以内
- 数据呈现：核心数据+一句话解读
- 风险提示：极简风险词
- 调整点：固定「一句话观点+话题标签」结构

【确认】风格手册已定稿
【重新学习】调整原始素材后重新分析

```

- 结尾格式：{描述}

## 词汇参数

## 句式参数

### 固化资源复制

**【必须执行】** 从固化资源池复制合规红线到输出目录：

### 自测评验证

**【必须执行】** 保存前运行自测评脚本：

```bash
python3 {skill_dir}/skills/style-learning/scripts/self_eval.py \
    --style-dir {skill_dir}
```

- 整体通过 → 继续保存
- 未通过 → 按报告修复 → 重新验证（最多3轮）
- 3轮未达标 → 标注警告，提示用户人工检查

## 持续学习：风格飞轮

### 学习我的修改

用户编辑文章后说「学习我的修改」：

1. 对比初稿 vs 用户修改稿
2. 提取新增风格特征
3. 更新 `style.yaml` + `exemplars/`
4. 提示：「已学习X处修改，风格档案已更新」

### 重新学习

用户说「重新学习风格」：

1. 清空 `references/exemplars/`
2. 从 Phase 1 重新执行
3. 提示：「已重新学习，最新风格档案已更新」

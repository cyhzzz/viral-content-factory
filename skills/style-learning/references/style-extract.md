## 8维度深度提取（财经场景）

### 维度1：标题格式（权重20%）

**提取内容**：
- 主标题结构（符号规范、字数范围、创作方法）
- 副标题格式（如"——本栏目为xxx观点收评"）
- 栏目名称（如"年年有鱼·市场评论"）
- 标题中的固定元素（如【】包裹的栏目名）

**提取格式**：
```markdown
## 标题格式
- 主标题：【{栏目}·{类型}】{核心观点}，{字数}字
- 副标题：——{格式}，{字数}字
- 栏目名：{栏目名}
- 标题符号：【】包裹栏目名，——引导副标题
```

---

### 维度2：段落结构（权重20%）

**提取内容**：
- 段落数量与排序
- 每段功能定位（第1段市场概述、第2段事件分析...）
- 段落间衔接逻辑
- 段落长度范围

**提取格式**：
```markdown
## 段落结构
- 段落数量：{N}段
- 各段功能：
  - 第1段：{功能}，「{固定开头语}」，{字数范围}
  - 第2段：{功能}，「{固定开头语}」，{字数范围}
  - ...
- 段落衔接：{衔接逻辑描述}
- 总字数：{min}-{max}字
```

---

### 维度3：句式特征（权重15%）

**提取内容**：
- 平均句子长度
- 条件判断句式（若...则...）
- 概率性表达（大概率、或、可能）
- 标点使用规律（分号、冒号使用频率）

**提取格式**：
```markdown
## 句式特征
- 平均句长：{N}字
- 条件判断句：{出现频率}，「{示例}」
- 概率表达：{出现频率}，「{示例}」
- 标点特点：{描述}
```

---

### 维度4：词汇特征（权重15%）

**提取内容**：
- 专业术语库（从样本中高频提取）
- 特色比喻词（如"鱼群""压舱石"等）
- 谨慎词使用（大概率、或、可能、建议关注）

**提取格式**：
```markdown
## 词汇特征
- 专业术语：{列表}
- 比喻词：
  - {比喻词}：指代{含义}（{出现频率}）
  - {比喻词}：指代{含义}（{出现频率}）
- 谨慎词：{列表}
- 禁用词：{列表}
```

```

## 11维度深度提取（小红书场景）

### 维度9：段落强制开头语

**提取内容**：

## 段落强制开头语

|:---:|:---|
| 第1段 | {如无则填"无"} |

| 第3段 | {如"操作思路上"} |
```

---

### 维度10：比喻体系

**提取内容**：
从样本中提取所有比喻词，确定其指代含义，区分"必须使用"和"可选使用"。

**提取格式**：
```markdown
## 比喻体系
- 核心比喻（必须使用）：
  - {比喻词}：指代{含义}
  - {比喻词}：指代{含义}
- 辅助比喻（可选）：
  - {比喻词}：指代{含义}
```

## 提取操作指南

### 逐篇分析

1. **逐篇提取**：先逐篇提取，再交叉对比
2. **重点标注**：对维度5（分析逻辑）投入更多精力
3. **示例保留**：每个维度都要保留具体示例

### 交叉比对

1. 识别跨文本的**稳定模式**（风格常量）
2. 识别因文而异的**弹性策略**（风格变量，主题适配特征）
3. 将"稳定模式"提炼为可描述、可模仿的风格内核

### 提取输出格式

```markdown
# 8+3维度提取报告

## 维度1：标题格式
{详细提取结果 + 具体示例}

## 维度2：段落结构
{详细提取结果 + 具体示例}

...

## 维度9：段落强制开头语
{详细提取结果}

## 维度10：比喻体系
{详细提取结果}

## 维度11：图片使用
{详细提取结果}
```

## 内容标准（最低门槛）

提取结果写入 `style_manual.md` 时必须满足：

| 文件 | 最低字数 | 说明 |
|------|---------|------|
| `references/style_manual.md` | ≥500字 | 通用风格手册整体内容 |
| `references/platform_styles/*.md` | ≥200字/文件 | 每个平台手册 |

**执行方式**：提取完成后统计字符数，不足则扩充该维度的描述和示例。

## IP名称一致性规则（按平台）

**核心原则**：IF 提取到了 → 全文件一致；IF 未提取到 → 跳过。

### 需要检查的元素

| 元素 | 规则 |
|------|------|
| IP自称 | 提取到 → 各平台手册一致；未提取 → 不检查 |
| 品牌口号 | 提取到 → 全文件一致；未提取 → 不检查 |
| 固定结尾模板 | 提取到 → 各平台手册一致；未提取 → 不检查 |
| CTA句式 | 提取到 → 各平台手册一致；未提取 → 不检查 |

### 占位符规则（任何时候都不得有）

所有 `{}` 占位符必须全部替换为实际内容，无论是否提取到对应元素。

### 各平台IP自称差异

同一IP在不同平台可能使用不同自称（如微信用"Xavier"，小红书用"Xav"）。检查规则：
- 通用风格手册记录 IP 的多个自称（按平台）
- 各平台手册使用该平台对应的自称
- 跨平台自称一致性仅在同一自称被提取时检查

### 确认流程

```
风格档案生成后 → 按平台分别确认
    │
    ├── references/style_manual.md    → 通用部分整体确认
    ├── platform_styles/wechat.md     → 微信自称一致？
    ├── platform_styles/xhs.md        → 小红书自称一致？
    ├── platform_styles/weibo.md      → 微博自称一致？
    └── ...（其他平台同理）
```

## 轻量提取模式（1-2篇）

当可用文章仅 1-2 篇时，执行轻量提取：

### 提取原则

- **只提取能明确识别的维度**：凡是需要跨文本对比才能确认的模式，跳过
- **未明确的维度标注 `[待补充]`**：留作后续手动补充，不回退到任何预设人格参数
- **不降级使用预设人格**：人格仅作为最末 fallback（用户拒绝提供任何文章时）

### 操作步骤

1. 对可用文章逐篇提取（不做交叉对比）
2. 每维度判断：能明确提取 → 写入；无法确认 → 标注 `[待补充]`
3. 标记优先补充维度（维度 5 分析逻辑、维度 7 固定表达优先）

### 标注格式

```markdown
## 维度3：句式特征
- 平均句长：[待补充]
- 条件判断句：[待补充]
- 概率表达：[待补充]
- 标点特点：[待补充]
```

### 后续补充

当用户提供更多文章后，执行「补充提取」：
- 仅补充 `[待补充]` 的维度
- 已确认的维度不重新提取（以首次提取为准）

# 格式整理规则

> 本文档是拆解改写流程 Step 1R.2 的详细操作手册

---

## 格式整理目标

将非结构化的原始文本，转化为结构清晰的 Markdown，保留原文全部信息，不做内容删改。

---

## 格式整理流程

### 第一步：全文扫描

1. 读取原文，统计全文字数
2. 识别文章标题（首行或 `<h1>` / `#` 标记）
3. 识别段落数量（空行分隔）
4. 识别是否存在标题层级（H2/H3标记）
5. 识别数据表格、代码块、引用块

### 第二步：段落边界识别

**段落类型判断**：

| 类型 | 识别特征 | 处理方式 |
|:---:|:---|:---|
| 标题段 | 短句，无句末标点或加粗 | `#` 或 `##` |
| 正文段 | 含完整句子，有句末标点 | 保持原样 |
| 列表段 | 含序号/项目符号 | 转为 `-` 或 `1.` |
| 引用段 | 以 `>` 开头 | 保持 `>` |
| 数据表格 | 行列结构，含分隔符 | 转为 Markdown 表格 |
| 代码段 | 含缩进或```标记 | 保持缩进或``` |
| 图片占位 | 含 `[图片]` 或 `![]` | 标注 `[图片：描述]` |

### 第三步：标题层级重建

**规则**：

1. **已有标题层级**（原文有 `#` 或 `##` 标记）：
   - 保留原文标题层级
   - 检查是否有遗漏的子章节，手动补充

2. **无标题层级**（原文为纯段落）：
   - 按内容逻辑切分章节
   - 根据每段首句判断是否为章节开头
   - 章节数建议 3-6 个（太少则章节跨度大，太多则碎片化）

**章节切分判断**：

| 判断依据 | 示例 |
|---------|------|
| 话题转换 | "从资金面来看..." 标志新话题 |
| 强转折词 | "然而"、"但是"、"不过" 标志转折 |
| 数字序列 | "第一..." "第二..." 标志并列结构 |
| 时间序列 | "先是..." "然后..." "最后..." 标志时间线 |
| 段落位置 | 文章前1/3多为背景，中间为核心，后1/3为结论 |

### 第四步：特殊元素标注

**数据块识别**：

| 原文特征 | 标注方式 |
|---------|---------|
| 含数字+%/倍/元等 | `[数据：{摘要}]` 标注在段内 |
| 含 "根据XX数据显示" | 保持原文，标注 `[数据来源]` |
| 含排名/榜单 | 转为 Markdown 表格 |

**引用块识别**：

| 原文特征 | 处理方式 |
|---------|---------|
| 以引号开头 | 保留引号，加 `> ` |
| 含 "XX说：" | 保留引用格式，标注说话人 |
| 含 "如图所示" | 标注 `[引用位置]` |

**图片占位**：

| 原文特征 | 标注方式 |
|---------|---------|
| 含 "如图X所示" | `[图片：图X描述]` |
| 含图片URL | `![图片描述](url)` |
| 无图片但提到图 | `[图片：原文描述]` |

## 格式整理输出规范

### 结构化 Markdown 要求

1. **保留原文全部文字**，不做任何删改
2. **仅调整格式**：添加标题层级、标注特殊元素
3. **数据保持精确**：数字、百分比、金额原文保留
4. **标点原文保留**：不统一标点风格

### 格式整理示例

**输入（原文）**：
```
昨日A股市场呈现分化格局。上证指数微涨0.09%，深证成指下跌0.35%，创业板指下跌1.08%。两市成交额约2.15万亿元，较前一日缩量约300亿元。

从资金面来看，主力资金净流出超过200亿元。其中，饮料制造板块净流出居首，超过30亿元。

市场人士分析认为，节前资金面偏紧是主要原因。若成交量不能有效放大，突破前高难度较大。
```

**输出（结构化Markdown）**：
```markdown
## 市场概述

昨日A股市场呈现分化格局。上证指数微涨0.09%，深证成指下跌0.35%，创业板指下跌1.08%。两市成交额约2.15万亿元，较前一日缩量约300亿元。

## 资金面分析

从资金面来看，主力资金净流出超过200亿元。其中，饮料制造板块净流出居首，超过30亿元。

## 后市展望

市场人士分析认为，节前资金面偏紧是主要原因。若成交量不能有效放大，突破前高难度较大。
```

## 常见问题处理

### 问题1：原文含大量HTML标签

**处理方式**：
1. 去除 HTML 标签，保留标签内文本
2. `<strong>` 内文本保留并加粗
3. `<p>` 段落转换为 Markdown 空行分隔

### 问题2：原文为微信外链形式

**处理方式**：
- 外链转为上标编号脚注形式：
  ```markdown
  这是正文内容[1]。

  ---

  [1] 参考链接：https://...
  ```

### 问题3：原文含 emoji

**处理方式**：
- 保留原文 emoji，不做转换
- 确认终端支持显示

### 问题4：原文含中英混排

**处理方式**：
- 不强制加空格
- 保留原文

### 问题5：原文含特殊符号（★ ◆等）

**处理方式**：
- 保留原文
- 不转为其他符号

## 格式整理检查清单

完成整理后，逐项核对：

- [ ] 全文字数与原文一致（字数差异 < 5%为可接受误差）
- [ ] 所有段落内容完整，无遗漏
- [ ] 标题层级逻辑清晰（同级标题并列，不同级标题有包含关系）
- [ ] 数据原文保留（数字/百分比/金额）
- [ ] 引用块正确标注
- [ ] 无乱码或无法识别的字符
- [ ] 原文无删改（仅格式调整）

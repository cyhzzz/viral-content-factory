# 审校流程

## 核心定位

对初稿进行三维度质量把关，在进入多平台改写前确保内容质量合格。

---

## 审校三维度

| 维度 | 内容 | 输出格式 |
|------|------|---------|
| SEO优化 | 标题策略/摘要/关键词/标签 | 3个备选标题 + 5个标签 |
| 内容质量 | 深入浅出/吸引力/情绪弧线 | 问题清单 + 修改建议 |
| 合规审校 | Error/Warning/Info 三级 | 问题定位 + 修改建议 + 修订版全文 |

---

## Step 5.1 SEO优化

```
读取: skills/review/references/seo-rules.md
```

### SEO输出要求

1. **3个备选标题**：在原标题基础上优化，或重新创作
2. **摘要**：≤40字，含核心关键词
3. **5个标签**：#话题标签格式

### 标题优化规则

- 主标题15-30字
- 含核心关键词
- 有悬念/情绪/数字元素
- 适合平台分发

---

## Step 5.2 内容质量审校

```
读取: skills/review/references/quality-check.md
```

### 审校维度

| 维度 | 评估内容 |
|------|---------|
| 深入浅出 | 是否说人话、小白是否读得懂、专业术语是否解释 |
| 吸引力 | 开头是否抓人、信息密度、是否有干货感 |
| 情绪弧线 | 是否有起伏、有无高潮设计、结尾是否有力量 |

### 审校输出

每维度：
- **评估**：通过 / 有问题
- **问题**：具体描述（定位到段/句）
- **建议**：具体可执行的修改建议

---

## Step 5.3 合规审校

```
读取: skills/review/references/compliance.md
```

### 合规Severity分级

| 级别 | 符号 | 含义 | 处理 |
|------|------|------|------|
| Error | 🔴 | 必须修改才能发布 | 标红，用户确认修改 |
| Warning | 🟡 | 表述过于极端 | 标黄，给出建议，用户可选保留 |
| Info | 🔵 | 建议优化项 | 标蓝，供参考，不阻断 |

### 合规检查项

| 检查项 | Error | Warning |
|--------|-------|---------|
| 提及个股 | 直接提及个股名称 | — |
| 收益承诺 | "一定涨"/"保证赚钱" | — |
| 极端表述 | "暴跌"/"崩盘" | "大概率"/"可能"缺失 |
| 风险提示 | 无风险提示 | 不完整 |
| 数据来源 | 无来源数据 | 来源模糊 |

### 合规修改原则

- 最小修改：以词语替换/删除为主
- 保留风格：修改尽量不影响原文的语气和节奏
- 逐条标注：每条问题对应原文具体位置

---

## Step 5.4 修订版全文

### 生成规则

1. 读取原始初稿
2. 逐条应用 Error 修改建议
3. 保留 Warning（标注供用户选择）
4. 忽略 Info（仅供参照）
5. 输出融合所有修改后的完整版本

### 输出格式

```markdown
# 修订版全文

{完整文章，含所有Error修改}
```

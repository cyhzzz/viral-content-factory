# 内容类型框架路由

## 核心定位

根据选题的内容类型，将写作任务路由到对应的框架文件，返回完整的段落骨架和写作规范。

## 内容类型定义

| 类型 | 适用场景 | 核心特征 |
|------|---------|---------|
| 市场评论 | 每日/每周市场走势分析 | 数据驱动、事件跟踪、盘面解读 |
| 热点解读 | 突发新闻、行业政策解读 | 新闻切入、影响分析、趋势预判 |
| 行业分析 | 板块/概念/产业链深度分析 | 产业逻辑、竞争格局、长期趋势 |
| 投教营销 | 基础知识普及、工具介绍、权益说明 | 通俗易懂、行动引导、转化导向 |

## 路由规则

### 路由入口

读取 `{skill_dir}/references/topic-selection.md` 中的选题评分结果，或根据用户原始输入判断。

### 自动路由判断

| 关键词特征 | 匹配类型 |
|-----------|---------|
| 今日市场、收盘、上证、深证、创业板、行情、板块、资金 | 市场评论 |
| 突发、刚刚、政策发布、事件解读、新闻 | 热点解读 |
| 行业、板块、产业链、竞争格局、龙头、ETF、投资逻辑 | 行业分析 |
| 怎么入门、新手、什么是、有什么用、如何开通、介绍 | 投教营销 |
| 解读、分析、影响、深度 | 按内容实质判断 |

无明确关键词时：
- 看涨跌幅/成交量数据 → 市场评论
- 看是否有明确新闻事件 → 热点解读
- 看是否有产业链/竞争分析 → 行业分析
- 看是否为教学/科普向 → 投教营销

### 路由确认

- 全自动模式 → 直接路由，不停顿
- 交互模式 → 展示判断结果，请用户确认类型选择

## 框架文件结构规范

每个框架文件必须包含以下章节：

### 1. 框架概述

```markdown
## {类型名称}

**适用场景**：{一句话说明}
**核心特征**：{3-5个关键词}
**篇幅建议**：{字数范围}
**内容类型路由权重**：{Primary / Secondary}
```

### 2. 段落骨架

```markdown
## 段落结构

| 段落 | 功能 | 固定开头语 | 字数范围 |
|:---:|:---|:---|:---:|
| 第1段 | {功能} | {固定开头语} | {min-max}字 |
| ... | ... | ... | ... |
```

### 3. 各段落写作规范

每个段落需详细说明：
- **本段核心任务**：2-3句话
- **必备要素**：列出必须包含的元素
- **可选要素**：列出可选择包含的元素
- **金句方向**：建议的金句落点方向
- **数据要求**：需要哪些类型的数据

### 4. 标题规范

```markdown
## 标题规范

**结构**：[符号规范][栏目名·类型]核心观点
**字数**：{min}-{max}字
**情绪**：{冷静/犀利/温暖/警示}
**示例**：
- 【示例标题1】
- 【示例标题2】
```

### 5. 数据使用规范

```markdown
## 数据使用规范

**必需数据**：
- {数据类型1}：{精度要求}，{表达方式}
- {数据类型2}：{精度要求}，{表达方式}

**可选数据**：
- {数据类型}

**数据来源标注格式**：
- {标注格式}
```

### 6. 句式规范

```markdown
## 句式规范

**必须使用的句式**：
- 条件判断句：{示例}
- 概率性表达：{示例}

**推荐使用的句式**：
- {句式类型}：{示例}

**禁止使用的句式**：
- ❌ {禁止表达}
```

### 7. 词汇规范

```markdown
## 词汇规范

**高频专业术语**：
- {术语1}：{含义}
- {术语2}：{含义}

**谨慎使用词**：
- ✅ {推荐词}
- ❌ {禁用词}

**特色词汇（如有）**：
- {比喻词}：指代{含义}
```

### 8. 风险提示规范

```markdown
## 风险提示规范

**位置**：{文章开头/结尾/两者皆有}
**格式**：
```
{风险提示原文}
```

**合规检查项**：
- [ ] {检查项1}
- [ ] {检查项2}
```

### 9. 质量检查清单

```markdown
## 质量检查清单

初稿完成后逐项核对：

**结构层面**：
- [ ] {检查项}

**内容层面**：
- [ ] {检查项}

**合规层面**：
- [ ] {检查项}
```
